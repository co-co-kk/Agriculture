# 组件拆分与注释规范

## 1. 目录分层
- `src/pages`：页面组装层
- `src/components/layout`：布局组件
- `src/components/scene-disease`：病害场景组件
- `src/components/scene-pest`：虫害场景组件
- `src/components/common`：通用组件
- `src/components/map`：地图组件
- `src/components/charts`：图表组件
- `src/hooks`：数据与回放逻辑
- `src/stores`：状态管理
- `src/mocks`：mock 生成、选择器、handlers

## 2. 组件化规则
1. 页面层仅做组装，不写复杂业务逻辑。
2. 每个组件只负责单一职责。
3. 超过 300 行必须拆分。
4. 状态逻辑放 `store/hooks`，不放在深层展示组件。

## 3. 中文注释规则
1. 每个代码文件顶部必须有“文件职责注释”。
2. 每个导出函数/组件必须有中文注释。
3. 复杂计算、条件分支、联动逻辑必须写中文块注释。
4. 关键 JSX 区块建议写结构注释。
5. 禁止无意义注释，注释需解释“为什么/做什么”。

## 4. 验收清单
1. 随机抽查代码文件，确认存在中文职责注释。
2. 随机抽查导出函数，确认存在中文说明。
3. 检查单文件长度，不超过 300 行。
4. 检查页面逻辑是否按分层目录组织。
