# 农业AI双场景作战看板 PRD

## 1. 项目目标
构建一个仅前端的农业 AI 平台演示系统，覆盖病害识别与虫害检测两类无人机巡检场景，重点展示真实平台感、交互联动能力与可扩展的数据结构。

## 2. 用户与使用场景
- 目标用户：农业企业管理者、农技人员、平台演示观众。
- 使用方式：在统一看板内切换场景，查看地图热区、识别结果、告警与处置建议。

## 3. 范围
- 包含：统一看板、双场景切换、地图热区、影像识别框、告警流、报告抽屉、mock 数据接口化。
- 不包含：真实模型推理、真实后端、真实无人机设备接入。

## 4. 页面结构
1. 顶部状态栏：场景切换、KPI、刷新、报告入口。
2. 左侧分析栏：筛选面板 + 场景统计图表。
3. 中部地图栏：地块热力、任务轨迹、当前帧影像识别。
4. 右侧详情栏：告警流、高风险区域、处置建议。
5. 底部时间轴：任务选择、帧拖动、自动回放。

## 5. 关键能力
1. 场景1：真菌/细菌/病毒感染识别，支持严重度筛选。
2. 场景2：害虫种类、数量、分布检测，支持虫种筛选。
3. 地图联动：点击地块或告警可定位并高亮区域。
4. 回放联动：拖动时间轴可刷新帧图与检测结果。
5. 报告导出：支持导出当前场景 JSON 报告。

## 6. 验收标准
1. 双场景切换无报错，数据与图表同步刷新。
2. 时间轴拖动时帧信息和识别框变化明显。
3. 地图可点击地块并与右侧区域列表联动。
4. 告警列表可点击定位地块。
5. 报告抽屉可打开并成功导出 JSON。

## 7. 非功能要求
1. 所有代码文件含中文注释。
2. 单文件不超过 300 行。
3. 组件职责单一，禁止页面内堆积业务逻辑。
